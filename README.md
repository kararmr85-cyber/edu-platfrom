# منصة الطالب العراقي - السادس الإعدادي

أساس متكامل وقابل للعمل فعليًا لمنصة تعليمية باللغة العربية (RTL) لطلاب السادس الإعدادي
في العراق، بفرعيها العلمي والأدبي.

## ✅ ما تم بناؤه فعليًا في هذا الإصدار (يعمل من البداية للنهاية)

- **قاعدة بيانات كاملة** (`supabase/schema.sql`): كل الجداول، العلاقات، RLS، Triggers،
  ودوال محفزة (XP، المستوى، إنشاء بروفايل تلقائي عند التسجيل) + بيانات أولية للمواد والإنجازات.
- **المصادقة**: تسجيل حساب، تسجيل دخول، تسجيل خروج، إعادة تعيين كلمة المرور — متكاملة مع Supabase Auth.
- **نظام الاختبارات الكامل** (الأولوية التي طلبتها):
  - `POST /api/quiz/start` يسحب أسئلة عشوائية من بنك الأسئلة حسب المادة (ويدعم تصفية الأسئلة الوزارية فقط).
  - صفحة أداء الاختبار (`/quiz/[subjectId]`) فيها مؤقّت تنازلي حقيقي، تصفّح بين الأسئلة، شريط تقدّم،
    ودعم لكل أنواع الأسئلة (اختيار من متعدد، صح/خطأ، إكمال فراغ، شرح، تعريف).
  - `POST /api/quiz/submit` يصحّح الإجابات تلقائيًا، يحسب النقاط والنسبة، يمنح XP، يرفع المستوى،
    ويمنح الإنجازات تلقائيًا.
  - صفحة نتائج تفصيلية (`/quiz/[subjectId]/result`) تعرض كل سؤال وإجابتك والإجابة الصحيحة.
  - `GET /api/quiz/leaderboard` لوحة صدارة حسب نقاط الخبرة.
- **لوحة تحكم الطالب**: تتبع التقدم، عدد الاختبارات، متوسط الدرجات، آخر النشاطات، XP والمستوى.
- **لوحة تحكم الأدمن**: نظرة عامة بإحصائيات حقيقية + صفحة فعلية لإضافة أسئلة لبنك الأسئلة.
- **صفحة الموارد**: عرض ملخصات وملفات PDF مرتبطة بكل مادة (الرفع الفعلي يحتاج Supabase Storage - راجع الخطوات التالية).
- **الصفحة الرئيسية**: تصميم عصري، متجاوب، Dark Mode حقيقي يعمل بدون وميض، عربي RTL بالكامل.
- **Middleware** لحماية مسارات `/dashboard`, `/quiz`, `/admin` وتحديث الجلسة تلقائيًا.

## 🚧 ما يحتاج إكمالاً (هيكلية جاهزة، تحتاج صفحات فعلية)

- صفحة إدارة المستخدمين الكاملة في الأدمن (`/admin/users`)
- صفحة إدارة المواد والوحدات في الأدمن (`/admin/subjects`)
- رفع PDF فعلي إلى Supabase Storage (`/admin/resources`) + API route للرفع
- نموذج "إضافة مادة" و"إضافة وحدة" في الأدمن
- صفحة `subjects-list` API المستخدمة في `admin/questions` (مرجع سريع غير منشأ بعد)
- تحسين تصحيح أسئلة "الشرح/التعريف" (حاليًا تصحيح تقريبي بالنص، يفضل مراجعة بشرية أو AI)
- صفحة تأكيد إعادة تعيين كلمة المرور (`/reset-password/confirm`)

## التشغيل محليًا

\`\`\`bash
npm install
cp .env.example .env.local   # ثم ضع بيانات مشروع Supabase الخاص بك
\`\`\`

1. أنشئ مشروع جديد في [supabase.com](https://supabase.com)
2. شغّل محتوى `supabase/schema.sql` كاملاً في SQL Editor
3. فعّل Email Auth في Authentication > Providers
4. أضف الروابط في `.env.local`

\`\`\`bash
npm run dev
\`\`\`

## النشر على Vercel

1. ادفع المشروع إلى GitHub
2. استورده في [vercel.com](https://vercel.com)
3. أضف متغيرات البيئة نفسها من `.env.local` في إعدادات Vercel
4. اضغط Deploy

## بنية المشروع

\`\`\`
app/
  (auth)/login, signup, reset-password
  dashboard/         لوحة الطالب
  quiz/[subjectId]/  نظام الاختبارات + النتائج
  admin/             لوحة تحكم الأدمن
  api/quiz/          start, submit, leaderboard
  api/admin/         questions
components/          مكوّنات مشتركة (Navbar, ThemeToggle, ...)
lib/supabase/        عملاء Supabase (متصفح/خادم)
lib/types.ts         أنواع TypeScript مطابقة لقاعدة البيانات
supabase/schema.sql  قاعدة البيانات الكاملة
\`\`\`
